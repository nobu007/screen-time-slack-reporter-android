# PR-001: Implement Core Logic Unit Tests

## 概要
GAP-001 (Test coverage 0%) および GAP-002 (Webhook validation verified) を解消するため、
コアロジックに対するユニットテストを実装する。

## 目的
- `SlackWebhookValidator` のバリデーションロジックが正しく動作することを保証する (CF-005)
- `SendDailyReportUseCase` のビジネスロジックが期待通り動作することを保証する (CF-002)
- CIパイプラインでテストが実行され、通過する状態を作る (QA-001)

## 変更内容

### 1. バリデータテストの追加
- `app/src/test/java/jp/co/screentime/slackreporter/data/slack/SlackWebhookValidatorTest.kt` を作成
- 正常系: `https://hooks.slack.com/services/...` を許可
- 異常系: 不正なURL、空文字、異なるドメインを拒否

### 2. UseCaseテストの追加
- `app/src/test/java/jp/co/screentime/slackreporter/domain/usecase/SendDailyReportUseCaseTest.kt` を作成
- MockKを使用して依存コンポーネントをモック化
- 正常系: データ取得成功時にSlack送信が呼ばれること
- 異常系: データ取得失敗時やSlack送信失敗時の挙動確認

## 検証方法

### 自動テスト実行
```bash
./gradlew testDebugUnitTest
```

### 期待される結果
- 全テストが PASS する
- カバレッジレポート `app/build/reports/jacoco/jacocoTestReport/html/index.html` で対象クラスのカバレッジが上昇する
- `verify_core_functions.py` の `CF-005` 検証が PASS に変わる

## ロールバック手順
```bash
rm app/src/test/java/jp/co/screentime/slackreporter/data/slack/SlackWebhookValidatorTest.kt
```
